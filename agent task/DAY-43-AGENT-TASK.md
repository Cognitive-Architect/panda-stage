## 【模块1】饱和攻击头部（通用增强版）

- **火力配置**：1 Agent（Engineer）
- **任务名称**：Real Production Sample B + Efficiency Comparison
- **轰炸目标**：在不复制 Sample A 项目、不修改代码的前提下完成第二条同复杂度样片，复用角色定义与动作知识，记录同样的步骤耗时，并诚实比较哪些环节变快、没变或变慢。
- **任务性质**：真实生产验证 + 效率测量 + 复用价值判断
- **输入基线**：Day 42 实际 P0 修复已通过；Sample A 数据完整；RC/Gate C 仍有效；Sample B 脚本与素材权利清晰。
- **输出要求**：样片 B MP4 + 项目目录 + 同口径时间表 + A/B 对比 + 现成工具基准 + 未改善项 + `docs/validation/SAMPLE-B.md`。
- **通用铁律**：
  1. 样片 B 必须使用不同但同复杂度脚本，禁止为了数据好看降低难度。
  2. 必须从空项目开始，不能复制 Sample A 项目。
  3. 可以复用角色定义、素材和动作知识，但必须记录复用内容与准备成本。
  4. 制作过程中禁止修改代码或手改项目 JSON。
  5. 若没有变快，必须如实记录，不得挑选局部数字制造进步。

## 【模块3】工单矩阵

### 基础信息

- **工单编号**：B-43/45
- **角色**：Engineer
- **依赖关系**：依赖 Sample A 与 Day 42 修复结果。

### 输出交付物

- **必须产出**：
  - 一条约 30 秒样片 B MP4；
  - 完整项目目录；
  - 脚本和素材许可清单；
  - 与 Sample A 同字段的步骤耗时表；
  - 复杂度对照表；
  - 复用内容与复用准备成本；
  - A/B 绝对时间和百分比变化；
  - 现成工具同口径基准或 N/A + 原因；
  - 变快、没变、变慢的步骤；
  - `docs/validation/SAMPLE-B.md`。
- **制作流程**：脚本/素材→空项目→导入→角色→镜头→摆位→动作→对白字幕音频→预览→保存重开→导出检查。
- **必须包含**：
  - Sample A/B 时长、镜头、角色、对白、动作规模对照；
  - 不能复制 Sample A 项目目录或 JSON；
  - 明确哪些角色定义/素材允许复用；
  - 复用准备时间计入或单独列出；
  - 使用 Sample A 同样的主动操作/等待/返工口径；
  - 全程不改代码/JSON；
  - 保存重开后一致；
  - 导出 MP4 经 ffprobe 验证；
  - 每一步计算 A/B 时间变化；
  - 将学习熟练度与 Day 42 修复收益分开讨论；
  - 没有改善的步骤诚实保留；
  - 基准工具名称、版本、流程和证据明确；若无法公平比较则写 N/A。
- **禁止包含**：
  - 使用更短或更简单脚本；
  - 复制 Sample A 项目；
  - 制作途中改代码；
  - 只比较总时长而忽略规模差异；
  - 删除返工时间；
  - 用假设值填现成工具基准；
  - 只挑变快项目、不列变慢项目。
- **交付证明**：项目和 MP4、ffprobe、全程录屏、时间日志、复杂度对照、A/B 计算表、基准证据与 SAMPLE-B 文档。

### 规模与复杂度观察

- 样片复杂度不可能完全相同，因此必须使用镜头数、对白数、动作数等指标解释差异。
- “效率提升”至少同时给绝对分钟数与百分比，且说明样本只有两条，不能过度外推。
- 复用价值需要把“创建一次、后续复用”的成本分开，避免把素材准备免费化。
- 现成工具基准若无法真实执行，不得凭印象补数字。

## 【模块3-B】地狱红线

1. Day 42 阻塞修复未通过仍执行 → 停止。
2. 复制 Sample A 项目 → 数据失效。
3. 使用更简单脚本 → 数据失效。
4. 制作途中改代码/JSON → 任务违规。
5. 计时口径与 Sample A 不一致 → 返工。
6. 删除返工或等待时间 → 数据造假。
7. 只列变快、不列没变或变慢 → 返工。
8. 外部基准使用猜测值 → 返工。
9. 未导出/ffprobe 就声称完成 → 未验证。
10. 过度外推两条样本 → 结论失真。
