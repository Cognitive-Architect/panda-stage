## 【模块1】饱和攻击头部（通用增强版）

- **火力配置**：1 Agent（Engineer）
- **任务名称**：Project Portability + Recovery + Repeated Export Resilience
- **轰炸目标**：对 RC 执行项目改名/移动、中文路径、崩溃恢复、10 次短导出、10 次开始取消、缺素材、损坏音频、磁盘空间不足等测试，并记录资源与清理结果。
- **任务性质**：稳定性验证 + 故障注入 + 资源观察
- **输入基线**：Sample A/B 已完成；Day 42 修复仍通过；RC/Gate C 关键路径稳定；测试项目和短导出 fixture 可用。
- **输出要求**：可复现韧性矩阵 + 10 次导出/取消数据 + 内存/进程/temp 报告 + 错误可读性 + `docs/test-receipts/RESILIENCE.md`。
- **通用铁律**：
  1. 每次测试必须记录前置、操作、预期、实际和证据路径。
  2. 内存、进程和临时文件结论必须来自真实采样，禁止凭感觉写“无泄漏”。
  3. 故障注入不得破坏用户原始项目，必须使用副本和隔离目录。
  4. 连续测试中发现 P0 时立即停止扩展压力，保留现场并记录。
  5. 所有未验证项必须明确列出，不能用一次成功替代重复验证。

### 基础信息

- **工单编号**：B-44/45
- **角色**：Engineer
- **依赖关系**：依赖 Sample A/B 与最新 RC 修复基线。

### 输出交付物

- **必须产出**：
  - 项目改名测试；
  - 项目目录移动测试；
  - 中文用户名与中文空格路径测试；
  - 异常退出与 recovery 测试；
  - 连续 10 次短项目完整导出；
  - 连续 10 次开始→取消；
  - 缺图片/背景素材测试；
  - 损坏音频测试；
  - 磁盘空间不足或安全等价故障测试；
  - 每轮峰值/结束内存、FFmpeg/Electron 残留、temp 数量与大小；
  - `docs/test-receipts/RESILIENCE.md`。
- **必须包含**：
  - 所有测试使用项目副本；
  - 移动/改名后通过重新定位打开，内部相对素材仍有效；
  - 最近项目失效项可处理；
  - recovery 内容与异常退出前 autosave 一致；
  - 10 次导出使用同一短项目和输出设置；
  - 每次导出产物可播放并记录 hash/大小/时长；
  - 内存记录基线、峰值、结束值和趋势；
  - 10 次取消发生在不同或明确相同阶段并记录完成时间；
  - 每次取消后无 FFmpeg/隐藏窗口/temp/半成品残留；
  - 缺素材和坏音频指出具体文件与下一步；
  - 磁盘不足不破坏项目或成功产物；
  - 故障后 UI 可再次导出；
  - 全部未验证项单列；
  - 发现 P0 时保留日志和复现，不继续用后续循环覆盖现场。
- **禁止包含**：
  - 在真实唯一项目上做破坏性测试；
  - 只跑一次就声称稳定；
  - 用任务管理器单个瞬时数值断言无泄漏；
  - 取消后不检查进程和文件；
  - 为通过测试手工清理后不记录；
  - 隐瞒某轮失败；
  - 顺手开发新功能或大重构。
- **交付证明**：测试矩阵、PowerShell/系统进程与内存记录、目录前后快照、10 次结果表、错误截图、recovery 对照、MP4/ffprobe 与回执。

### 规模与复杂度观察

- 自动化脚本仅负责重复操作和采样，不替代真实桌面行为；脚本必须可单独运行和清理。
- “无明显泄漏”需要定义判断口径，例如多轮结束内存不持续单调上升、进程和临时目录回到稳定范围；不能随意给百分比阈值。
- 磁盘不足优先使用受控小容量虚拟磁盘/配额或 FileSystemService 故障注入，不得填满系统盘。
- 如果环境无法安全模拟磁盘不足，写 `N/A + 原因 + 替代故障注入 + DEBT-TEST-B44-001`。

## 【模块3-B】地狱红线

1. 使用唯一真实项目做破坏性测试 → 严重违规。
2. 只执行一次就声称韧性通过 → 返工。
3. 内存结论无采样记录 → 未验证。
4. 取消后不查进程/temp → 未验证。
5. 手工清理残留却不记录 → 数据造假。
6. 填满系统盘模拟磁盘不足 → 危险违规。
7. 隐瞒某轮失败 → 严重违规。
8. 发现 P0 后继续覆盖现场 → 返工。
9. 顺手扩功能或大重构 → 范围失控。
10. 未验证项未列出仍判通过 → 返工。
