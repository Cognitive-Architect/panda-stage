# Panda Stage Agent Task — Day 09

> **工单编号**：B-09/45  
> **角色**：Engineer  
> **来源**：`DAILY_PLAN.md` Day 9  
> **分支建议**：`spike/day-09-path-cancel-cleanup`  
> **任务类型**：稳定性强化 + Windows 路径验证  
> **唯一目标**：让完整探针在中文与空格路径下成功导出，并实现渲染、写盘、编码阶段的统一取消与资源清理。

---

## 【模块1】饱和攻击头部（通用增强版）

- **火力配置**：1 Agent（Engineer）
- **任务名称**：Unicode 路径 + 导出取消 + 资源清理
- **轰炸目标**：统一 Export Job 的取消信号，使隐藏窗口、待写队列、当前编码子进程和 UI 状态都能停止并恢复，同时验证 Windows 中文、空格、Unicode 路径。
- **任务性质**：稳定性强化 + 负面路径验证
- **输入基线**：Day 6～8 已形成帧捕获、静音视频编码和单音轨 AAC 合成链路。
- **输出要求**：中文路径完整导出 + 连续 5 次开始/取消证据 + 无残留窗口、子进程和临时目录 + 结构化收卷。
- **通用铁律**：
  1. 取消必须停止真实后台工作，不是只改变界面状态。
  2. 只能终止当前 Export Job 持有的编码子进程，不得影响其他应用或其他任务。
  3. 路径处理使用参数数组与 Node 路径 API，不依赖手工添加引号。
  4. 清理逻辑必须幂等，重复调用不会产生新故障。
  5. 取消后必须可以立即重新开始一次完整导出。

## 【模块3】工单矩阵（通用高压版）

### 1）基础信息

- **工单编号**：B-09/45
- **角色**：Engineer
- **目标**：完成 Unicode 路径支持、统一取消传播与彻底清理。
- **依赖关系**：依赖 Day 6～8 完整探针链路；无并行依赖。

### 2）输出交付物

- **预计变更文件**：
  - `electron/main/services/ExportService.ts`
  - `electron/main/services/FFmpegAdapter.ts`
  - `electron/main/services/FileSystemService.ts`
  - `electron/main/ipc/` 下 export/cancel handler
  - `src/export-window.tsx`
  - 必要测试文件
  - `docs/test-receipts/DAY-09.md`
- **核心修改点**：
  - Job 状态加入 `cancelling` / `cancelled` 或等价状态；
  - 取消信号传播到隐藏窗口帧循环；
  - 停止新的帧请求和写队列入队；
  - 等待当前写入句柄释放后清理；
  - 只终止当前 Job 对应的编码子进程；
  - 删除临时目录与半成品；
  - UI 恢复可再次导出状态；
  - 所有路径通过 Node 路径 API 和参数数组传递。
- **必须包含**：
  - 渲染阶段取消；
  - 编码阶段取消；
  - Unicode 项目和输出路径；
  - 连续 5 次开始→取消；
  - 取消后立即成功导出；
  - 错误信息包含 Job ID 和用户可读原因。
- **禁止包含**：
  - 系统级全局进程终止；
  - 仅隐藏 UI 而后台继续；
  - 路径字符串手工加引号；
  - 静默忽略临时目录删除失败；
  - 暂停/恢复、任务队列、多并发导出。
- **交付证明**：5 次取消日志、当前任务资源释放证据、临时目录清理证据、Unicode 路径输出和取消后再次成功导出。

### 3）规模与复杂度观察

- 取消链路是跨层状态机，每层只能释放自己持有的资源。
- 清理函数必须幂等；如 Windows 文件占用需要有限重试，必须说明次数与等待策略，禁止无限循环。
- 若平台限制要求对当前子进程采用特殊终止方式，必须限定到该 Job 并声明 `DEBT-PLATFORM-B09-001`。

### 4）自动化质量闸门

| 闸门 | 要求 | 验证命令 / 证据 | 不通过后果 |
|---|---|---|---|
| BUILD | 构建通过 | `pnpm build` | 返工 |
| TYPE | 类型检查通过 | `pnpm typecheck` | 返工 |
| FMT | 格式通过 | `pnpm exec prettier --check .` 或 N/A + 原因 | 返工或声明 |
| LINT | 无新增 lint error | `pnpm lint` | 返工 |
| TEST | 取消状态、幂等清理、路径处理测试通过 | `pnpm test:unit` + 适用集成测试 | 返工 |
| ARCH | 无系统级全局进程终止，无 shell 拼接 | diff 审查 + 静态搜索 | 返工 |
| REAL | 后台任务确实停止 | 当前 Job 日志、临时目录与子进程证据 | 返工 |
| DOC | Day 9 回执和协议同步 | 文档 diff | 返工或声明债务 |

## 【模块3-B】地狱红线（10 项）

1. 取消只改 UI，后台继续运行 → 返工。
2. 取消当前 Job 时影响其他应用或其他任务 → 返工。
3. Unicode 路径失败却改用英文目录绕过 → 返工。
4. 临时目录删除失败被静默吞掉 → 返工。
5. 取消后不能再次导出仍声称完成 → 返工。
6. 路径通过手工引号拼接 → 返工。
7. 幂等清理未验证 → 返工。
8. 顺手实现暂停、队列或多并发 → 范围失控。
9. 5 次循环没有真实证据 → 未验证。
10. 默认质量门禁失败仍交付 → 返工。
