## 【模块1】饱和攻击头部（通用增强版）

- **火力配置**：1 Agent（Engineer）
- **任务名称**：Real Production Sample A + Friction Log
- **轰炸目标**：把 RC 当成真正工具使用，从脚本、素材准备到成片导出完整走一遍，不在制作途中随手改代码，用数据暴露真实生产摩擦。
- **任务性质**：真实生产验证 + 流程测量 + 问题分级
- **输入基线**：RC1 必须 PASS；安装包、演示项目、导出、文档与已知问题可用；脚本和素材权利清晰。
- **输出要求**：约 30 秒样片 A + 项目目录 + 完整耗时表 + 失败/绕路表 + P0/P1/P2 分级 + 前三阻塞 + `docs/validation/SAMPLE-A.md`。
- **通用铁律**：
  1. RC1 非 PASS 时停止，不得进入真实生产验证。
  2. 样片制作过程中禁止修改代码、手改 `project.json` 或借助开发者控制台绕过 UI。
  3. 记录必须使用分钟、次数和具体步骤，禁止用“感觉慢”“有点麻烦”替代数据。
  4. 发现问题先记录，不在制作途中顺手全部修复。
  5. 样片脚本、图片、音频和字体必须原创、已获许可或权利清晰。

### 基础信息

- **工单编号**：B-41/45
- **角色**：Engineer
- **依赖关系**：依赖 RC1 PASS。

### 输出交付物

- **必须产出**：
  - 一条约 30 秒样片 A MP4；
  - 完整项目目录；
  - 脚本与素材许可清单；
  - 制作全程步骤与分钟数；
  - 每步失败次数、绕路和提示是否可理解；
  - 需要改代码的点；
  - P0/P1/P2 问题清单；
  - 前三个生产阻塞；
  - `docs/validation/SAMPLE-A.md`。
- **制作流程**：
  1. 固定脚本与目标时长；
  2. 准备背景、2 个角色和 6～8 句音频；
  3. 从空项目开始；
  4. 导入素材并创建角色；
  5. 创建镜头；
  6. 摆位、变换与动作；
  7. 添加对白、字幕、音频和嘴巴；
  8. 完整预览；
  9. 保存、关闭、重开；
  10. 导出并播放检查；
  11. 执行 ffprobe；
  12. 汇总摩擦与阻塞。
- **必须包含**：
  - 从空项目开始，不能复制 Sample 或 Demo；
  - 全程不改代码/JSON；
  - 每个步骤记录开始/结束与净耗时；
  - 记录等待、失败、返工和绕路次数；
  - 区分“软件问题”“素材问题”“操作不熟”；
  - 记录需要查文档或猜操作的位置；
  - 保存重开后内容一致；
  - 预览与导出关键表现一致；
  - ffprobe 验证编码与时长；
  - 问题按 P0/P1/P2 定义和证据分级；
  - 前三阻塞按生产影响排序；
  - 未触发的 RC1 已知问题也要记录为“未触发”，不能消失。
- **禁止包含**：
  - 制作途中修代码；
  - 手改项目 JSON；
  - 用演示项目替代；
  - 为数据好看删掉失败记录；
  - 用主观感受替代时间和次数；
  - 把操作不熟全部归咎软件；
  - 顺手扩功能或做视觉美化。
- **交付证明**：屏幕录制、项目目录、MP4、ffprobe JSON、逐步时间表、失败截图、问题证据和 SAMPLE-A 文档。

### 规模与复杂度观察

- 本日不追求开发产出，核心资产是“真实生产数据”。
- 计时口径必须统一：主动操作时间、等待时间、返工时间可分列，避免一个模糊总时长。
- P0 应定义为无法在不改代码的情况下完成核心成片；P1 为严重影响效率但有规避；P2 为体验问题。
- 若样片无法完成，仍须收卷为 FAIL，并记录阻塞点，不能换更简单脚本重来掩盖问题。

| 闸门 | 要求 | 验证命令 / 证据 | 后果 |

## 【模块3-B】地狱红线

1. RC1 非 PASS 仍执行 → 停止。
2. 制作途中修改代码 → 任务违规。
3. 手改 `project.json` 绕过 UI → 任务违规。
4. 用演示项目代替真实样片 → 返工。
5. 删除失败记录美化数据 → 严重违规。
6. 用“感觉慢”代替分钟数 → 返工。
7. 为数据好看换更简单脚本 → 返工。
8. 问题无证据却标 P0 → 返工。
9. 未导出/ffprobe 就声称完成 → 未验证。
10. SAMPLE-A 不完整仍收卷 → 返工。
